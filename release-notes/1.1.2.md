# CloakBrowser Desktop 1.1.2

## 更新概览

本版本重新发布安装包，重点修复安装协议中文乱码，并扩展 GitHub Actions 云端打包矩阵，新增 macOS 与 Linux 的多 CPU 架构安装包。

## 安装器修复

- 修复 Windows 安装器协议页中文显示乱码问题。
- 将安装前合规与免责协议从 UTF-8 文本改为 NSIS 可稳定读取的 RTF 协议文件。
- 保留交互式安装流程，用户仍需接受协议并可自行选择安装目录。

## 新增平台安装包

- Windows x64：NSIS 安装包。
- macOS x64：DMG 与 ZIP 安装包。
- macOS arm64：DMG 与 ZIP 安装包。
- Linux x64：AppImage 安装包。
- Linux arm64：AppImage 安装包。

## 发布流程改进

- Release workflow 改为矩阵打包，不同系统和 CPU 架构分别在对应 GitHub runner 上构建。
- Release 资产文件名增加平台和架构标识，避免不同架构产物互相覆盖。
- macOS 与 Linux 的更新元数据会按架构重命名后上传，避免 `latest-mac.yml`、`latest-linux.yml` 多架构冲突。

## 签名与安全说明

- Windows 代码签名证书入口仍保留，配置 `WINDOWS_CSC_LINK` 与 `WINDOWS_CSC_KEY_PASSWORD` 后可由云端打包流程自动签名。
- 当前如果未配置商业代码签名证书，浏览器或 Windows 仍可能提示“通常不会下载”或要求用户手动确认。
- GitHub Artifact Attestation 继续用于记录 Release 产物的构建来源。

## 安装说明

- Windows 用户请选择 `windows-x64` 安装包。
- Apple Silicon Mac 用户请选择 `mac-arm64` 安装包。
- Intel Mac 用户请选择 `mac-x64` 安装包。
- Linux x64 / arm64 用户请选择对应架构的 AppImage。

CloakBrowserDesktop 仅可用于合法、合规的学习、研究、测试或经授权场景。不得用于未授权访问、账号滥用、欺诈、攻击、规避平台规则或其他违法违规用途。
